## Lawful basis and transparency

<ul style="list-style: none; padding-left: 0;">

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    An information audit is conducted to determine what personal data is processed and who has access to it.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    Each data processing activity has a clear and documented legal basis.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    Clear information about data processing and its legal justification is provided in the privacy policy.
  </li>

</ul>

<ul style="list-style: none; padding-left: 0;">

  <li>
    <input type="checkbox" style="accent-color: #2e7d32;" disabled />
    A record of processing activities can be produced if required by regulators.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #2e7d32;" disabled />
    A data protection impact assessment (DPIA) is conducted when processing is likely to result in high risk.
  </li>

</ul>

<ul style="list-style: none; padding-left: 0;">

  <li>
    <input type="checkbox" style="accent-color: #2e7d32;" disabled />
    Records of processing include:
    <ul>
      <li>Purpose of processing</li>
      <li>Types of personal data processed</li>
      <li>Who has access to the data</li>
      <li>Third parties involved and their location</li>
      <li>Security measures (e.g. encryption)</li>
      <li>Planned data retention or deletion period</li>
    </ul>
  </li>

</ul>

## Data security

<ul style="list-style: none; padding-left: 0;">

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    Data protection is considered from the initial design phase and throughout all processing activities.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    Personal data is encrypted, pseudonymized, or anonymized whenever possible.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    An internal security and data protection policy exists for team members.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    Team members are trained and made aware of data protection responsibilities.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    There is a documented process for conducting data protection impact assessments.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    A breach notification process exists to inform authorities and affected users when required.
  </li>

</ul>

<ul style="list-style: none; padding-left: 0;">

  <li>
    <input type="checkbox" style="accent-color: #2e7d32;" disabled />
    Data protection by design and by default is applied using appropriate technical and organizational measures.
  </li>

</ul>

## Accountability and governance

<ul style="list-style: none; padding-left: 0;">

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    A responsible person is designated to oversee GDPR compliance.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    Data processing agreements are in place with third parties processing personal data.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    An EU representative is appointed if the organization operates outside the EU.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    A Data Protection Officer (DPO) is appointed where legally required.
  </li>

</ul>

<ul style="list-style: none; padding-left: 0;">

  <li>
    <input type="checkbox" style="accent-color: #2e7d32;" disabled />
    Accountability for GDPR compliance is clearly defined and enforced within the organization.
  </li>

</ul>

## Privacy rights

<ul style="list-style: none; padding-left: 0;">

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    It is easy for customers to request and receive all personal data stored about them.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    It is easy for customers to correct or update inaccurate or incomplete personal information.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    It is easy for customers to request deletion of their personal data.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    It is easy for customers to request that processing of their personal data stops.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    It is easy for customers to receive a copy of their personal data in a portable format.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    It is easy for customers to object to the processing of their personal data.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #ed6c02;" />
    If automated decision-making is used, procedures exist to protect user rights.
  </li>

</ul>

### Right of access – transparency & compliance

<ul style="list-style: none; padding-left: 0;">

  <li>
    <input type="checkbox" style="accent-color: #2e7d32;" />
    Users can see what personal data is stored and how it is used.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #2e7d32;" />
    Users are informed how long their data is retained and why.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #2e7d32;" />
    The first copy of personal data is provided free of charge.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #2e7d32;" />
    Reasonable fees may apply for additional copies.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #2e7d32;" />
    Identity verification is performed before releasing personal data.
  </li>

  <li>
    <input type="checkbox" style="accent-color: #2e7d32;" />
    Requests can be fulfilled within one month.
  </li>

</ul>
